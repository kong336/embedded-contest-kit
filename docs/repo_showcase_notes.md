# Repository Showcase Notes

This page is about presentation rather than firmware logic.

## What makes this repository good for a profile

- It is narrow and focused instead of being a mixed personal dump.
- It has reusable code, examples, docs, and local verification.
- It shows both embedded fundamentals and engineering organization.

## Best things to show first

If someone opens the repository, the strongest things to point them to are:

1. `README.md`
2. `docs/contest_fast_start.md`
3. `docs/project_recipes.md`
4. `examples/contest_app_template.c`
5. `stm32_hal/examples/stm32_contest_app_template_hal.c`
